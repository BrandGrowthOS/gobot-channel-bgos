01

Slash picker — force-reseed when stuck

shouldSeedDefaults() only fires when command_count === 0. On older BGOS deploys whoami doesn't return the field at all → the seed never runs, picker stays empty.

Before
/ |
No slash commands
Configure under Integrations → Assistant
After
/ |
  • /rememberSave a fact to memory
  • /trackTrack a new goal with optional deadline
  • /doneMark a goal as completed
  • /forgetForget a stored fact
  • /cancelCancel an in-flight goal
  • /criticAsk the critic agent for a devil's advocate take
  • /boardRun a multi-agent board meeting on a topic
How the fix works 
# Before: shouldSeedDefaults(undefined) returns false → no seed.
# After: new GOBOT_BGOS_RESEED_COMMANDS env knob + a CLI you can run on demand.

$ npx gobot-bgos-reseed-commands
✔ assistant=42 reseeded 7 commands
✔ assistant=43 reseeded 7 commands

# Or set in your launchd plist / VPS env:
$ GOBOT_BGOS_RESEED_COMMANDS=safe bun run src/bot.ts
  • auto — seed only when count is 0 (default — safe for prod)
  • safe — also seed when count is missing (covers older deploys)
  • always — force every startup (emergency fix)
  • never — manage exclusively from Integrations UI
02

/board — six agents, one bound assistant, six labeled bubbles

The fork's runBoardMeeting already fans out via seven Telegram bot identities. In BGOS there's only one bound assistant — but the backend already supports from_agent_inline. We stamp each agent's bubble with their name + accent color so the thread reads as a multi-voice debate, not a wall of generic assistant text.

async function runMorningBriefing() {
  const channel = resolveHomeChannel(); // reads GOBOT_HOME_CHANNEL

  if (channel !== "bgos") {
    await sendTelegramMessage(BOT_TOKEN, CHAT_ID, briefing);
  }
  if (channel !== "telegram") {
    await sendProactiveToBgos(briefing);
  }
}

Test plan after deploy

  1. Slash picker — open BGOS, focus the composer for your bound assistant, type /. Picker shows the 7 Gobot commands. If empty, run npx gobot-bgos-reseed-commands.
  2. /board run — send /board Should we launch a paid newsletter? Expect the orchestrator announcement, 7 agent bubbles (Research, Content, Finance, Strategy, CTO, COO, Critic — each with its own name + accent color), and one synthesis bubble. All from the single bound assistant.
  3. /track — send /track Test goal | tomorrow. Goal saved to memory; assistant confirms inline.
  4. Morning briefing — wait for the next launchd briefing trigger (or run bun run briefing manually). With GOBOT_HOME_CHANNEL=both, the briefing arrives in Telegram AND in your BGOS Ava chat. With =bgos, BGOS only.
